# Kubernetes

"Model" Kubernetes configurations and command references.

- `models/` → versioned sets of related manifests that represent common patterns (stateless service, job, etc.)
- `commands/` → useful kubectl + ecosystem commands with explanations

See root [docs/naming-and-versioning.md](../docs/naming-and-versioning.md) for how versions work here (K8s API versions and best practices change over time).
