# Python

Reference models and structures for Python projects.

## Models

See `models/` for versioned project models.

Current focus:
- Modern packaging (pyproject.toml + src layout)
- Clear separation of application code and tests
- Practical defaults that work well with uv, ruff, mypy, pytest, etc.

## How to use

1. Pick a model (e.g. `web-api`).
2. Copy the desired version directory.
3. Rename the package/module as needed.
4. Read the version's `README.md` for initialization steps and rationale.

## Compatibility Notes

Models in this section generally target Python 3.11+ unless otherwise stated in the individual model README.
